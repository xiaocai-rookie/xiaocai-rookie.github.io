问题产生的原因

​ 在一次开发中甲方要求，需要做一个自动旋转的菜单，并且鼠标放在菜单中旋转停止在当前位置。

​ 接需求是内心是开心的，因为刚学习了 transform 的 rotate属性 ，刚好就能用上，心里是相当的开心的。

​ 做好第一版是这样的 示例(左边的样子) ,然后 我是崩溃的，为什么调整了z-index 这货不按套路出牌呢，完全没用

​ 这个时候就需要百度+Google了，好的，万能的百度网优告诉我，这是由于层叠层级混乱，出现了多个层叠空间引起的

​ 啊？这是什么问题，从来没见过。好的，请看下文

问题：

​ 什么是层叠空间？

​ 层叠空间对页面有什么影响?

​ 什么情况会造成多个层叠空间?

造成原因

​ 层叠空间，官方说法是层叠上下文，我们知道在页面布局时会用到定位属性，如relative、absolute、inherit、fixed、static

  • ``static` 默认属性，产生定位效果
  • relative 相对于上一个具有定位能力的父元素进行定位且不会脱离正常文档流，若没有父元素满足这个条件则取html为基准元素
  • absolute 相对于上一个具有定位能力的父元素进行定位且会脱离正常文档流，若没有父元素满足这个条件则取html为基准元素
  • fixed 永远相对于窗口位置定位
  • inherit 规定应该从父元素继承 position 属性的值

在正常文档流中，relative、absolute整个页面文档会是一个大的 定位空间，大定位空间包裹小定位空间，当小定位空间失效时会往父元素查询，寻找可以进行相对定位的父元素，然后相对于找到的父元素进行定位；如果一直没有找到可以定位的父元素，则以html作为相对定位的父元素

然而当元素中出现以下css属性时，则会产生独立的层叠空间，不会与其他层叠空间产生互动的层叠效果且z-index属性调整不同的层叠空间的层叠顺序时无效(z-index 属性只能调整属于同一层叠空间的层叠属性)，仅会遵守正常文档流中后来居上的层叠顺序进行展示

  • transform
  • filter
  • perspective
  • clip-path
  • mask / mask-image / mask-border

以上解释了我的为什么会出现的问题，即transfrom:rotate 时 ，每一个元素下都产生了一个独立的层叠空间，且层叠空间与层叠空间之间仅遵守 文档初始化时的 顺序，无法通过z-index属性改变展示顺序

解决

最开始时 我尝试的方法是，层叠空间内的内容进行互换，保证内容为当前的内容就好，但是一想好复杂啊，有没有简单的方法呢

后来想到一个办法，新建一个div 放在菜单列表的最外面且是渲染的最后，这样就负荷 多个层叠空间后来居上的展示逻辑
